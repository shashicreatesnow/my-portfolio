# Projects Folder

This folder is the **content workspace** for all portfolio case studies. It lives outside the app code on purpose — it's for planning, writing, and organizing before anything goes into the CMS.

## Folder Structure

```
projects/
├── _instructions/       ← You are here. Guides and templates.
├── agentic-design/      ← AI-powered workflows, agentic tools, automation
├── ui-ux-design/        ← Product design, web apps, dashboards, user flows
├── visual-design/       ← Posters, thumbnails, visual systems, illustrations
├── brand-design/        ← Logos, brand identity, brand systems
```

## How to Use

1. **One markdown file per project.** Name it clearly: `project-name.md`
2. **Write freely.** Dump notes, context, links, raw thoughts — Claude will help shape it into a proper case study.
3. **Add research files** (inspiration, references, competitor screenshots) alongside the markdown.
4. **When a case study is ready**, the content gets pushed into the portfolio CMS through the admin panel.

## What Goes in Each Project File

At minimum, include:
- **Project name** and client (or personal project)
- **What you did** — your role and contribution
- **The problem** — what needed solving
- **The outcome** — what changed, any numbers or feedback
- **Assets** — links to Figma files, images, Behance/Dribbble, etc.

Don't worry about polish. Raw and honest is better than empty and perfect.

| Pillar | What it signals to visitors |
